## 📋 Deployment to Render (Recommended - 5 minutes)

### Step 1: Push code to GitHub
```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/YOUR_USERNAME/kyc-app.git
git push -u origin main
```

### Step 2: Create Render Account
1. Go to https://render.com
2. Sign up (free)
3. Connect your GitHub account

### Step 3: Deploy
1. Click "Create +" → "Web Service"
2. Connect your repository
3. Configure:
   - **Name:** kyc-backend
   - **Build Command:** `cd kyc_backend && pip install -r requirements.txt && python manage.py collectstatic --noinput && python manage.py migrate && python manage.py seed_data`
   - **Start Command:** `cd kyc_backend && gunicorn kyc_project.wsgi:application --log-file -`
   - **Environment:** Python 3.11
   - **Plan:** Free

### Step 4: Set Environment Variables in Render
In Render dashboard, go to your service → Environment:

```
USE_POSTGRES=true
DEBUG=false
SECRET_KEY=<AUTO-GENERATED - leave as is>
DATABASE_URL=<AUTO-LINKED - Render handles this>
CORS_ALLOWED_ORIGINS=https://YOUR-RENDER-DOMAIN.onrender.com
```

### Step 5: Deploy Frontend
1. Click "Create +" → "Static Site"
2. Connect your repository (same one)
3. Configure:
   - **Build Command:** `cd kyc_frontend && npm install && npm run build`
   - **Publish Directory:** `kyc_frontend/dist`
   - **Environment Variables:**
     ```
     VITE_API_URL=https://kyc-backend.onrender.com/api/v1
     ```

### Step 6: Get Your URLs
- **Backend:** https://kyc-backend.onrender.com
- **Frontend:** https://kyc-frontend.onrender.com

## 🔄 Alternative: Deployment to Railway

### Step 1: Push to GitHub (same as above)

### Step 2: Create Railway Account
Go to https://railway.app and sign up with GitHub

### Step 3: New Project → GitHub Repo
1. Click "Create New Project"
2. Select "Deploy from GitHub repo"
3. Choose your repository

### Step 4: Auto-detection
Railway will automatically detect Django and PostgreSQL

### Step 5: Set Variables
Go to Variables tab:
```
USE_POSTGRES=true
DEBUG=false
SECRET_KEY=<generate-strong-random-string>
CORS_ALLOWED_ORIGINS=YOUR-DOMAIN
```

### Step 6: Deploy
Railway auto-deploys. Check logs for deployment status.

## 🆘 Troubleshooting

### "ModuleNotFoundError: No module named 'django'"
- Build command didn't run properly
- Check Render/Railway logs for errors

### "ERROR: Database URL not set"
- Ensure `USE_POSTGRES=true` is set
- Render/Railway should auto-link PostgreSQL database
- Check if database service is running

### "CORS Error: Origin not allowed"
- Update `CORS_ALLOWED_ORIGINS` with your frontend domain
- Format: `https://frontend-domain.com`

### Frontend shows "Cannot connect to API"
- Check `VITE_API_URL` environment variable
- Should be: `https://your-backend-domain.com/api/v1`
- Verify backend is running and accessible

### Login not working
- Ensure seed data ran (check logs for "✅ TEST CREDENTIALS")
- Verify credentials against test data above
