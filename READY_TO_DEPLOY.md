## 🚀 Deploy Now (Choose One)

### Option 1: Render (Easiest - 5 minutes)
```bash
1. Go to render.com → Create Account (free with GitHub)
2. Create New → Web Service
3. Connect your GitHub repo
4. Auto-configure or use this build command:
   cd kyc_backend && pip install -r requirements.txt && python manage.py migrate && python manage.py seed_data
   
5. Start command: cd kyc_backend && gunicorn kyc_project.wsgi
6. Create → Done! Auto-deploys
7. Deploy frontend separately as Static Site
```

**URLs After Deployment:**
- Backend: `https://kyc-backend.onrender.com`
- Frontend: `https://kyc-frontend.onrender.com`

### Option 2: Railway (Also Easy - 5 minutes)
```bash
1. Go to railway.app → Sign in with GitHub
2. Create New Project → GitHub Repo
3. Railway auto-detects Django + PostgreSQL
4. Set environment variables (see below)
5. Deploy - auto-triggers on push
```

### Option 3: Fly.io
```bash
1. Install fly CLI: curl https://fly.io/install.sh | sh
2. fly auth login
3. fly launch (auto-generates fly.toml)
4. fly deploy
```

## 🎓 Testing Workflow on Deployment

1. **Login as Merchant1**
   ```
   merchant1 / merchant123
   ```

2. **Create KYC Submission**
   - Go to Dashboard → Create New
   - Fill in personal & business info
   - Save as draft

3. **Upload Documents**
   - Upload PAN (PDF/JPG/PNG, max 5MB)
   - Upload Aadhaar
   - Upload Bank Statement

4. **Submit for Review**
   - Click Submit → Goes to "submitted" status
   - Notification created
   - Appears in reviewer queue

5. **Login as Reviewer**
   ```
   reviewer1 / reviewer123
   ```

6. **Review Queue**
   - See all submitted submissions
   - Sorted by oldest first
   - Check SLA (at-risk count)

7. **Approve/Reject**
   - Open submission
   - Choose action: Approve / Reject / Request More Info
   - Add notes
   - Submission status updates
   - Merchant gets notification

8. **Test More Info Flow**
   - Merchant logs back in
   - Sees "more_info_requested" status
   - Can edit and resubmit
   - Goes back to "submitted" → reviewer queue

## 🆘 Troubleshooting

### Database not initializing
- Check logs: `python manage.py migrate`
- Render/Railway auto-run migrations from build command

### Seed data not created
- Check if `seed_data` command in build command
- Or manually run after deployment in platform shell

### Frontend can't reach API
- Verify `VITE_API_URL` is set correctly
- Check backend URL is accessible
- Run `curl https://your-backend/api/v1/` to test

### Login fails
- Ensure seed_data ran
- Check correct username/password from above
- Verify CORS allows your frontend domain
